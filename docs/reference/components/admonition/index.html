Admonition

Used to communicate a state that affects a system, feature, or page.

  • View on Github
  • View on Figma

import { getAdmonitionProps } from '@pluralsight/headless-styles'
interface AdmonitionOptions {
  sentiment?: 'info' | 'success' | 'warning' | 'danger'
}

getAdmonitionProps(options?: AdmonitionOptions)
import type { AdmonitionOptions } from '@pluralsight/headless-styles/types'

Anatomyâ€‹

  1. Container
  2. Status icon
  3. Title
  4. Description
  5. Close button (optional)
  6. Content button (optional)

}

Admonition Statusâ€‹

Info admonition

This is an example of an informational admonition.

Success admonition

This is an example of an success admonition.

Warning admonition

This is an example of an warning admonition.

Danger admonition

This is an example of an danger admonition.
  1. Info: Brings awareness to important information and sets expectations for the user.
  2. Success: Provides the user with positive confirmation of a successfully performed action or series of steps.
  3. Warning: Helps users avoid error situations and should communicate important or time-sensitive states of the system.
  4. Danger: Alerts of a problem that has already occurred, as a result of user actions.

Status Icon Mappingâ€‹

Icons recommended to use with each status.
const AdmonitionIconMap = {
  info: 'InfoCircleIcon',
  success: 'CheckCircleIcon',
  warning: 'WarningTriangleIcon',
  danger: 'DangerDiamondIcon',
}

Closable Admonitionâ€‹

To use an admonition with a close button, add in an IconButton.

function Admonition() {
  const admonition = getAdmonitionProps()
  const { button, iconOptions } = getIconButtonProps(
    admonition.iconButtonOptions
  )


  return (
    <div {...admonition.wrapper}>
      <span {...admonition.iconWrapper}>
        <InfoCircleIcon {...getIconProps(admonition.iconOptions)} />
      </span>
      <div {...admonition.textContainer}>
        <p {...admonition.title}>
          <strong>Closable Admonition</strong>
        </p>
        <small {...admonition.description}>
          This Admonition can be closed if you add a click handler to the
          button.
        </small>
      </div>
      <button {...button}>
        <CloseIcon {...getIconProps(iconOptions)} />
      </button>
    </div>
  )
}

Referenceâ€‹

getAdmonitionProps(options?: AdmonitionOptions)

Parametersâ€‹

  • options: A String value of a theme sentiment 'info' | 'success' | 'warning' | 'danger' which determines the styling color theme produced.

Returnsâ€‹

An Object matching the Anatomy containing className and a11y properties relevant to each element/section. Additionally, it returns a iconOptions proprety to pass into the Icon helper.

Behaviorâ€‹

Placementâ€‹

Do place at the top of the current content area, to capture the attention of users or be read as quickly as possible.

Don't place in the middle of a content area.

Do place at the beginning of a form.

Don't place in the middle or end of a form.

Status Iconsâ€‹

Do use the Icons we provide in the status section.

Don't use Icons outside of the examples we provide in the status section.

Close Buttonâ€‹

Do include the Close button when the Admonition is communicating a real-time response or status to the user.

Don't use the Close button when the Admonition is static, warning, or informative messages.

Accessibilityâ€‹

The Pando Admonition, Icon, and IconButton APIs combined with semantic HTML allow the Admonition to be fully accessible and screen-readable.
