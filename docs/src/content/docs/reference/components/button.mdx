## Usage

### Palette

Buttons come in 3 palettes: `action`, `neutral`, and `danger`. The default palette is `action`.

```jsx
import { Button } from '@pluralsight/react'

function ActionButton() {
  return <Button palette="action">Action</Button>
}
```

<br />
<Button palette="action">Action</Button>

```jsx
import { Button } from '@pluralsight/react'

function NeutralButton() {
  return <Button palette="neutral">Neutral</Button>
}
```

<br />
<Button palette="neutral">Neutral</Button>

```jsx
import { Button } from '@pluralsight/react'

function DangerButton() {
  return <Button palette="danger">Danger</Button>
}
```

<br />
<Button palette="danger">Danger</Button>

### Disabled

To disable a button, use the `disabled` prop, as in the native html button attribute.

```jsx
import { Button } from '@pluralsight/react'

function DisabledButton() {
  return <Button disabled>Disabled Button</Button>
}
```

<br />
<Button disabled>Disabled</Button>

### Outline Buttons

When you need a button that is not the primary focus of the page, use an outline button. To create an outline button, use the `usage` prop and set it to outline.

```jsx
import { Button } from '@pluralsight/react'

function OutlineButton() {
  return <Button usage="outline">Outline</Button>
}
```

<br />
<Button usage="outline">Outline</Button>

### Text Buttons

When you need a button that is not the primary or secondary focus of the page, use a text button. To create a text button, use the `usage` prop and set it to `text`.

```jsx
import { Button } from '@pluralsight/react'

function TextButton() {
  return <Button usage="text">Text</Button>
}
```

<br />
<Button usage="text">Text</Button>

### Button with icon

To display an Icon at the start or end of a button, use the `startIcon` or `endIcon` prop.

```jsx
import { MoonIcon } from '@pluralsight/icons'

function StartIcon() {
  return <Button startIcon={<MoonIcon />}>Start Icon</Button>
}

function EndIcon() {
  return <Button endIcon={<MoonIcon />}>End Icon</Button>
}
```

### Customizing

There are 3 ways to customize the Button component: using the button recipe, extending the recipe, or passing a className prop.

#### Button recipe

Pando makes use of the <Link href="https://panda-css.com/docs/concepts/recipes">recipe concept from Pandacss</Link>. This means that you can pull the Pando Button styles into any component that accepts classes or classNames, including html.

:::caution[Pitfall]
Recipes only generate css class names. This means that a11y concerns would have to be handled manually because they cannot be automatically included like in Pando's React components package.
:::

```jsx
import { button } from '@/styled-system/recipes'

function ButtonWithRecipe() {
  return <button class={button()}>Default Button recipe</button>
}
```

<br />
<button class={button()}>Default Button recipe</button>

There are a number of variants available for the button recipe:

1. **palette**: `"action" | "neutral" | "danger"`
1. **size**: `"md" | "lg"`
1. **usage**: `"text" | "outline" | "filled"`

```jsx
import { button } from 'styled-system/recipes'

function ButtonWithVariant() {
  return (
    <button class={button({ palette: 'danger', size: 'md', usage: 'outline' })}>
      Button recipe using variants
    </button>
  )
}
```

<button class={button({ palette: 'danger', size: 'md', usage: 'outline' })}>
  Button recipe using variants
</button>

#### Extending the Button recipe

Using <Link href="https://panda-css.com/docs/concepts/merging-styles#merging-config-recipe-and-style-object">the cx function from Pandacss</Link>, you can extend the Button recipe to extend Pando button styles.

```jsx
import { css, cx } from 'styled-system/css'
import { button } from 'styled-system/recipes'

const newClassName = cx(button(), css({ bg: 'Thistle', color: 'black' }))
```

<br />
<button class={cx(button(), css({ bg: 'Thistle', color: 'black' }))}>
  Button recipe extended
</button>

#### Passing a `className` prop

You can pass a className prop to the Button component to customize the Button. This is useful if your project uses <Link href="https://github.com/css-modules/css-modules">CSS Modules</Link> or a CSS-in-JS library like <Link href="https://emotion.sh/docs/introduction">Emotion</Link>.

```jsx
import customStyles from './customStyles.module.css'

function CustomButton(props) {
  return <Button className={customStyles.customButton}>{props.children}</Button>
}
```
