## Behaviors

### Placement

<div class={exampleBorder}>
  <p>Always place buttons side-by-side as long as there is space to do so.</p>
  <LiveButton componentString={examples.liveButtonExample} />
</div>

### Alignment

<div class={exampleBorder}>
  <p>Align buttons to the end edge of a container when using to prompt a user to navigate through a sequence, or when confirming an action (on the right for left-to-right languages and on the left for right-to-left languages).</p>
  <LiveButton componentString={examples.liveAlignmentExample} />
</div>

<div class={exampleBorder}>
  <p>For focused tasks and single-page actions, align at the start edge of the container.</p>
  <LiveForm componentString={examples.liveFocusedTaskExample} />
</div>

### Spacing

<div class={exampleBorder}>
  <p>Space buttons apart by 16px in all cases.</p>
  <LiveButton componentString={examples.liveButtonSpacingExample} />
</div>

### Loading buttons

<div class={exampleBorder}>
  <p>A button may indicate a loading process after submit. Always place the loading animation on the start edge of the button, and ensure the button is disabled to prevent errant activation.</p>

## Content guidance

### Microcopy

<ul class={css({ listStyleType: 'disc' })}>
  <li>Use sentence-case capitalization</li>
  <li>Do not use all caps or title case</li>
</ul>

### Content considerations

<ul class={css({ listStyleType: 'disc' })}>
  <li>Lead with action verbs to tell the user what will happen next - It is especially important to ensure the action being performed is clear in Danger sentiment buttons</li>
  <li>Use concise, easy to scan, clear labels to indicate the action to the user</li>
  <li>Do not use long unnecessary phrases in button labels</li>
  <li>Use active voice to indicate the action</li>
  <li>Do not use vague or generic labels such as "yes" or "no"</li>
</ul>

### Accessibility: Expected actions for assistive technologies

<ul class={css({ listStyleType: 'disc' })}>
  <li>Ensure your label will make sense when spoken aloud with screen readers</li>
  <li>Prioritize using validation and errors over disabled buttons as they do not provide feedback</li>
</ul>
