## Behaviors

### Placement

<div class={exampleBorder}>
  <p>Always place buttons side-by-side as long as there is space to do so.</p>
  <LiveButton componentString={liveButtonExample} />
</div>

<div class={exampleBorder}>
  <p>Icon buttons should lead other buttons in order in a row.</p>
  <LiveIconButton componentString={liveIconButtonExample} />
</div>

### Alignment

<div class={exampleBorder}>
  <p>Align buttons to the end edge of a container when using to prompt a user to navigate through a sequence, or when confirming an action (on the right for left-to-right languages and on the left for right-to-left languages).</p>
  <LiveButton componentString={liveAlignmentExample} />
</div>

<div class={exampleBorder}>
  <p>For focused tasks and single-page actions, align at the start edge of the container.</p>
  <LiveForm componentString={liveFocusedTaskExample} />
</div>

### Spacing

<div class={exampleBorder}>
  <p>Space buttons apart by 16px in all cases.</p>
  <LiveButton componentString={liveButtonSpacingExample} />
</div>

<div class={exampleBorder}>
  <p>
    Always use icon buttons for icon actions, to ensure proper tap target sizing and spacing. <span class={css({ color: 'warning.text.200' })}>icon-only/text</span> icon buttons can use 8px separation in condensed areas
  </p>
  <LiveIconButton componentString={liveIconButtonSpacingExample} />
</div>
