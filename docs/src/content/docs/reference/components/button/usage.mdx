## Spec

Always "hug contents" button widths, relative to their label.

<ul class={css({ listStyleType: 'disc' })}>
  <li>Provides another level of hierarchy besides color, type, and sentiment</li>
  <li>A larger primary button is good for usability ("Add to cart" will be larger than "Edit" or "Cancel")</li>
  <li>Minimum or fixed width buttons will potentially lead to more container problems, especially in sidebars or mobile screens</li>
  <li>Lastly, and least importantly, it's easier on designers and developers to remember and implement</li>
</ul>

**Exceptions:**

<ul class={css({ listStyleType: 'disc' })}>
  <li>Stacking buttons: these should be the same width</li>
  <li>Buttons on mobile screens - stretch to fill the viewport on very small screens (less than 400px)</li>
  <li>Button labels need to be a minimum character set (minimum of ~5-8 characters). No "OK" buttons allowed.</li>
</ul>

### Size

Buttons come in two sized: `md` and `lg`. The default size is `lg`. Use consistent button sizes appropriate to their surroundings and use.

<LiveButton componentString={`<Button size='lg'>Large</Button>`} />

Use large buttons as the default in most cases. These present the largest tap targets and offer the most space for longer labels

token: <code>size='lg'</code>

<LiveButton componentString={`<Button size='md'>Medium</Button>`} />

token: <code>size='md'</code>

### Start and end icons

Use leading icons for presentational and ornamental glyphs only, to help communicate meaning and context.

<LiveButton componentString={`<Button startIcon={<EnvelopeIcon />}>Subscribe</Button>`} />

Directional or action-oriented icons such as arrows should always be placed on the end side of the button.

<LiveButton componentString={`<Button endIcon={<ArrowRightIcon />}>Next step</Button>`} />

Avoid using icons on both the start and end sides of a button. This has potential to confuse the user. Start and end icons can be used in some cases, i.e. an external link icon or new window icon to help convery these behaviors to the user.

<LiveButton componentString={`<Button startIcon={<PlusIcon />} endIcon={<WindowsIcon />}>Add new record</Button>`} />

<LiveButton componentString={`<Button startIcon={<PencilIcon />} endIcon={<ExternalLinkIcon />}>Edit in Photoshop</Button>`} />
