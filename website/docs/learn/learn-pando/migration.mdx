title: Migrating from Classic
---

Coming from our Classic system? Pando is completely different with it's own ecosystem and far beyond the reach of our old system. Having said that, this means Pando **is not and will never be a 1:1 mirror to Classic**.

Classic was never built for a multi-product company and just scaled poorly. Pando resolves both of those issues.

:::tip Need a migration walk-through?

If you are looking for a walk-through approach of migrating to Pando from Classic, check out our [Add Pando to a Website](../get-started/installation/add-to-website.mdx) page.

:::

## Breaking Changes

Here is a list of some breaking changes in Pando from Classic:

### Layout

| Classic API  | Pando Equivalent | Notes                     |
| ------------ | ---------------- | ------------------------- |
| App Frame    | Headless-styles  | Grid API                  |
| Aside        | Headless-styles  | Grid API                  |
| Equal Col.   | Headless-styles  | Grid API                  |
| Page Heading | Headless-styles  | Built into the CSS setup. |
| Page Width   | :x:              |                           |

### Utilities

:::danger Pitfall

Pando does not support motion or spacing tokens. There is enough studies that prove token usage of these properties deprecates performance when loading web experiences.

:::
