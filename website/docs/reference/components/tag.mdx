## Usage

### Basic Tag

```jsx live
function BasicTags() {
  const tagProps = getTagProps()

  return (
    <div
      data-site-override="alignGridCenter"
      {...getGridProps({ cols: 12, gap: 16 })}
    >
      <div {...getGridItemProps({ colSpan: 1 })}>
        <a href="#" {...getTagProps({ size: 's' }).tag}>
          small
        </a>
      </div>

      <div {...getGridItemProps({ colSpan: 2 })}>
        <a href="#" {...tagProps.tag}>
          default
        </a>
      </div>

      <div {...getGridItemProps({ colSpan: 2 })}>
        <a href="#" {...tagProps.tag}>
          <PlaceholderIcon {...getIconProps(getTagProps.iconOptions)} />
          default
        </a>
      </div>
    </div>
  )
}
```

#### Size Mapping

```typescript title="Tag size rules."
{
  s: 'For smaller/tighter experiences where the default size (m) cannot be supported.',
  m: 'Should be used in most, if not all cases.'
}
```

## Reference

```typescript
getTagProps(options?: TagOptions)
```

### Parameters

- `options`: A String value of either `'s' | 'm'` to display the size styles.

### Returns

An Object matching the [Anatomy](#anatomy) containing `className` and a11y properties relevant to each element/section. Additionally, it returns a `iconOptions` Object with to use as the parameters for the Icon API.

## Behavoir

### Usage

**Do** use the Tag when you need to link to a category/content.

**Don't** use a Tag without a link to a category. Instead use a Badge.

**Do** use the Pando Icons within the Tag for visual interest.

**Don't** use emoji's within the Tag.

**Do** use in moderation to prevent too much visual noise.

**Don't** exceed more than 3 rows of Tags within a page.

## Accessbility

The Pando Tag, and Icon APIs combined with semantic HTML allow the Tag to be fully accessible and screen-readable.
