## Usage

### Filled Badges

```jsx live
function FilledBadges() {
  const gridItem = getGridItemProps({ colSpan: 2 })

  return (
    <div
      {...getGridProps({ cols: 12, gap: 16 })}
      data-site-override="marginBottom"
    >
      <div {...gridItem}>
        <span {...getBadgeProps().badge}>default filled</span>
      </div>

      <div {...gridItem}>
        <span {...getBadgeProps({ sentiment: 'info' }).badge}>info filled</span>
      </div>

      <div {...gridItem}>
        <span {...getBadgeProps({ sentiment: 'success' }).badge}>
          success filled
        </span>
      </div>

      <div {...gridItem}>
        <span {...getBadgeProps({ sentiment: 'warning' }).badge}>
          warning filled
        </span>
      </div>

      <div {...gridItem}>
        <span {...getBadgeProps({ sentiment: 'danger' }).badge}>
          danger filled
        </span>
      </div>
    </div>
  )
}
```

### Badge with icon

```jsx live
function BadgeWithIcon() {
  const { badge, iconWrapper, iconOptions } = getBadgeProps()

  return (
    <div {...getGridProps({ cols: 12, gap: 16 })}>
      <div {...getGridItemProps({ colSpan: 2 })}>
        <span {...badge}>
          <span {...iconWrapper}>
            <PlaceholderIcon {...getIconProps(iconOptions)} />
          </span>
          Badge with Icon
        </span>
      </div>
    </div>
  )
}
```

### Sizes

```jsx live
function BadgeSize() {
  const gridItem = getGridItemProps({ colSpan: 1 })

  return (
    <div {...getGridProps({ cols: 12, gap: 16 })}>
      <div {...gridItem}>
        <span {...getBadgeProps({ size: 'xs' }).badge}>xs badge</span>
      </div>
      <div {...gridItem}>
        <span {...getBadgeProps().badge}>s (default)</span>
      </div>
    </div>
  )
}
```

#### Size Mapping

```typescript title="Badge size rules."
{
  xs: 'For smaller/tighter experiences where the default size (s) cannot be supported.',
  s: 'Should be used in most, if not all cases.'
}
```

## Reference

```typescript
getBadgeProps(options?: BadgeOptions)
```

### Parameters

```typescript
interface BadgeOptions {
  classNames?: string[]
  sentiment?: 'default' | 'info' | 'success' | 'warning' | 'danger'
  usage?: 'filled' | 'outline'
  size?: 'xs' | 's'
}
```

### Returns

1. An Object matching the [Anatomy](#anatomy) containing `className` and a11y properties relevant to each element/section including an unused class consisting of the naming pattern `pando-badge`.
2. An `iconOptions` Object to use as the parameters for the Icon API and an `iconWrapper` property to spread on the wrapper element containing the Icon.

## Behavior

### Types

**Do** use the filled Badge as the default use case for all badges.

**Don't** change the color of the Badge.

**Do** use the outline Badge as the "low priority" use case for a badge.

**Don't** use the outline Badge as the default use case.

### States

**Do** use the default size for all use cases where possible of the Badge.

**Don't** use the "xs" size _unless_ it is a **last resort**.

### Static/Action Scenarios

**Do** use the Badge as a non-interactive element.

**Don't** link to new pages, or use any other interactive actions for a Badge.

**Do** use the sentiment colors or a custom color option.

**Don't** use any color in the action sentiment theme category.

## Accessbility

The Pando Badge, and Icon APIs combined with semantic HTML allow the Badge to be fully accessible and screen-readable.
