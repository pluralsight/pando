## Anatomy

1. Container
2. Text (optional)
3. Select (optional)
4. Button Group (optional)
5. Button(s)

## Usage

### Show More Example

For user opt-in lazy-loading approach.

```jsx live
function ShowMoreButton(props) {
  return (
    <button {...getButtonProps().button} onClick={props.handleFetchMoreData}>
      Show more
    </button>
  )
}
```

### Newer/Older Example

For a modern pagination approach which displays the current data first.

```jsx live
function PaginationButtons(props) {
  const styles = getPaginationProps()

  return (
    <div {...styles.container}>
      <div>
        <button
          {...styles.newer}
          disabled={props.noNewData}
          onClick={props.handleFetchNewerData}
        >
          Newer
        </button>
        <button
          {...styles.older}
          disabled={props.noOldData}
          onClick={props.handleFetchOlderData}
        >
          Older
        </button>
      </div>
    </div>
  )
}
```

### Pagination Bar

For the allowing the most options to the user, but only ideal for Desktop layouts.

```jsx live
function PaginationBar() {
  const [rowOption, setRowOption] = useState('')
  const selectStyles = getSelectProps({
    id: 'row-count',
    name: 'row_count',
    placeholder: 'Choose row count',
    size: 'm',
    value: rowOption,
  })
  const styles = getPaginationProps({ cols: 3 })
  const iconProps = getIconProps(selectStyles.iconOptions)

  function handleSelectChange(e) {
    setRowOption(e.target.value)
  }

  return (
    <div
      {...styles.container}
      style={{
        alignItems: 'baseline',
        gridTemplateColumns: '1fr 1fr auto',
      }}
    >
      <div>
        <small {...styles.text}>
          <strong>1-25</strong> of 1,2345
        </small>
      </div>
      <div>
        <div {...selectStyles.fieldWrapper}>
          <div data-docsIgnore="iconSpacing" {...selectStyles.selectWrapper}>
            <select
              {...selectStyles.select}
              onChange={handleSelectChange}
              style={{ marginTop: 'initial' }}
            >
              <option value="5">5 Rows</option>
              <option value="10">10 Rows</option>
              <option value="25">25 Rows</option>
            </select>
            <span {...selectStyles.iconWrapper}>
              <ChevronDownIcon {...iconProps} />
            </span>
          </div>
        </div>
      </div>
      <div>
        <button {...styles.newer} disabled>
          Newer
        </button>
        <button {...styles.older}>Older</button>
      </div>
    </div>
  )
}
```

## Reference

```typescript
getPaginationProps(options?: PaginationOptions)
```

### Parameters

- `options`: A String value for a cols property to determine style rules for the Pagination elements.

### Returns

An Object matching the [Anatomy](#anatomy) containing `className` and a11y properties relevant to each element/section.

## Behavior

### Patterns

**Do** use the Show More Button if you have a simple list that can be lazy-loaded.

**Don't** use the Pagination Bar if you have small amounts of data.

**Do** use the newer/older Buttons if you have a multi-page list.

**Don't** use the newer/older Buttons if you have a small list.

**Do** use the Pagination Bar for a desktop layout.

**Don't** use the Pagination Bar for a mobile layout.

## Accessbility

The Pando Pagination, and Button APIs combined with semantic HTML allow the Pagination to be fully accessible and screen-readable.
