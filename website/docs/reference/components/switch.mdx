## Usage

### Basic Switch

```jsx live
function BasicSwitch() {
  const [checked, setChecked] = useState(false)
  const switchProps = getSwitchProps({
    checked,
    id: 'notfications',
    name: 'add_notifications',
  })

  function handleChange() {
    setChecked((prev) => !prev)
  }

  return (
    <FormControlProvider>
      <div {...switchProps.wrapper}>
        <Label htmlFor={switchProps.input.id}>Sign up for notifications</Label>
        <label {...switchProps.switchContainer}>
          <input {...switchProps.input} onClick={handleChange} />
          <span {...switchProps.switchTrack}>
            <span {...switchProps.switchThumb} />
          </span>
        </label>
      </div>
    </FormControlProvider>
  )
}
```

### Sizes

```jsx live
function SwitchSizes() {
  const [checked, setChecked] = useState(false)
  const switchProps = getSwitchProps({
    checked,
    id: 'notfications',
    name: 'add_notifications',
  })
  const smallSwitchProps = getSwitchProps({
    checked,
    id: 'notfications',
    name: 'add_notifications',
    size: 's',
  })

  function handleChange() {
    setChecked((prev) => !prev)
  }

  return (
    <form>
      <FormControlProvider>
        <div {...switchProps.wrapper}>
          <Label htmlFor={switchProps.input.id}>Default size</Label>
          <label {...switchProps.switchContainer}>
            <input {...switchProps.input} onClick={handleChange} />
            <span {...switchProps.switchTrack}>
              <span {...switchProps.switchThumb} />
            </span>
          </label>
        </div>
      </FormControlProvider>

      <FormControlProvider>
        <div {...smallSwitchProps.wrapper}>
          <Label htmlFor={smallSwitchProps.input.id}>Small size</Label>
          <label {...smallSwitchProps.switchContainer}>
            <input {...smallSwitchProps.input} onClick={handleChange} />
            <span {...smallSwitchProps.switchTrack}>
              <span {...smallSwitchProps.switchThumb} />
            </span>
          </label>
        </div>
      </FormControlProvider>
    </form>
  )
}
```

## Reference

```typescript
getSwitchProps(options?: SwitchOptions)
```

### Parameters

- `options`: An Object containing properties derived from the FormControl.fieldOptions results (disabled, invalid, readOnly, required) to determine the styling of the Switch state. An id and optional name property that accept a String value which connected Accessibility properties from the label to the Switch. A checked and optional indeterminate property that accept a Boolean value to determine the styling of the Switch state. A value property that accepts a String for form submission. An optional size property that accepts a String value of `'s' | 'm'` to determine the styling of the Switch elements relative to the size.

### Returns

An Object matching the [Anatomy](#anatomy) containing `className` and a11y properties relevant to each element/section.

## Behavior

### Layout

**Do** use multiple Switches in a vertical list.

**Don't** use multiple Switches in a single row.

**Do** provide a Label with a Switch.

**Don't** use a Switch inline without a Label. Instead use a hidden Label.

### Usage

**Do** use a Switch for giving users an way to opt-in to an optional feature.

**Don't** use a Switch for collecting consent. Instead us a [Checkbox](./checkbox.mdx).

## Accessibility

The Pando Switch API combined with semantic HTML allow the Switch to be fully accessible and screen-readable.
