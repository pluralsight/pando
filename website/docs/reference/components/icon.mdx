## Anatomy

1. Container (optional)
2. Icon

## Import

```javascript
import { getIconProps } from '@pluralsight/headless-styles'
```

```typescript
import type { IconOptions } from '@pluralsight/headless-styles'
```

## Usage

### Examples

```jsx live
function IconSizes() {
  const gridItem = getGridItemProps({ colSpan: 1 })

  return (
    <div
      data-site-override="alignGridCenter"
      {...getGridProps({ cols: 12, gap: 16 })}
    >
      <div {...gridItem}>
        <PlaceholderIcon {...getIconProps({ size: 's' })} />
      </div>
      <div {...gridItem}>
        <PlaceholderIcon {...getIconProps({ size: 'm' })} />
      </div>
      <div {...gridItem}>
        <PlaceholderIcon {...getIconProps({ size: 'l' })} />
      </div>

      <div {...getGridItemProps({ colSpan: 2 })}>
        <PlaceholderIcon {...getIconProps({ customSize: '3rem' })} />
      </div>
    </div>
  )
}
```

## Reference

```typescript
getIconProps(options?: IconOptions): ReturnProps
```

### Parameters

```typescript
interface IconOptions {
  ariaHidden?: boolean
  ariaLabel?: string
  classNames?: string[]
  customSize?: string
  size?: 's' | 'm' | 'l'
}
```

### Returns

1. An Object matching the [Anatomy](#anatomy) containing `className` and a11y properties relevant to the Icon element including an unused class consisting of the naming pattern `pando-icon`.

## Behavior

There are no behavior patterns required for the Icon.

## Accessibility

The Pando Icon API combined with semantic HTML allow the Icons to be fully accessible and screen-readable.
