## Usage

### Basic Buttons

```jsx live
function BasicIconButtons() {
  const gridItem = getGridItemProps({ colSpan: 1 })
  const actionBtn = getIconButtonProps({
    ariaLabel: 'action icon button example',
  })
  const defaultBtn = getIconButtonProps({
    ariaLabel: 'default icon button example',
    sentiment: 'default',
  })
  const dangerBtn = getIconButtonProps({
    ariaLabel: 'danger icon button example',
    sentiment: 'danger',
  })
  const disabledBtn = getIconButtonProps({
    ariaLabel: 'disabled icon button example',
    disabled: true,
  })

  return (
    <div {...getGridProps({ cols: 12, gap: 8 })}>
      <div {...gridItem}>
        <button {...actionBtn.button}>
          <PlaceholderIcon {...getIconProps(actionBtn.iconOptions)} />
        </button>
      </div>

      <div {...gridItem}>
        <button {...defaultBtn.button}>
          <PlaceholderIcon {...getIconProps(defaultBtn.iconOptions)} />
        </button>
      </div>

      <div {...gridItem}>
        <button {...dangerBtn.button}>
          <PlaceholderIcon {...getIconProps(dangerBtn.iconOptions)} />
        </button>
      </div>

      <div {...gridItem}>
        <button {...disabledBtn.button}>
          <PlaceholderIcon {...getIconProps(disabledBtn.iconOptions)} />
        </button>
      </div>
    </div>
  )
}
```

## Reference

```typescript
getIconButtonProps(options?: IconButtonOptions)
```

### Parameters

```typescript
interface IconButtonOptions {
  ariaLabel: string
  classNames?: string[]
  disabled?: boolean
  sentiment?: 'default' | 'action' | 'danger'
  usage?: 'square' | 'round' | 'text'
  size?: 'm' | 'l'
}
```

### Returns

1. An Object matching the [Anatomy](#anatomy) containing `className` and a11y properties relevant to each element/section including an unused class consisting of the naming pattern `pando-icon-btn`.
2. An `iconOptions` Object to use as the parameters for the Icon API.

## Behavior

### Patterns

**Do** use the Button for an actionable event needed from the user.

**Don't** use the Button for navigation or styled links. Instead use the TextLink component.

**Do** keep Button usage minimal within an area.

**Don't** use more than three Buttons in a single area.

**Do** space Buttons apart by 16px (1rem).

**Don't** space Buttons apart greater or less than 16px (1rem).

### Sentiment

**Do** use the "action" sentiment for all use cases where possible.

**Don't** use the "default" sentiment _unless_ it is a **last resort**.

**Do** use the "danger" sentiment for cancel or destructive actions.

**Don't** use the "default" sentiment for cancel or destructive actions.

### Usage

**Do** use the "outline" usage for a **lower priority alternative** use case to the "default" sentiment.

**Don't** use the "outline" usage for cancel or destructive actions.

**Do** use the "text" usage for the **lowest** priority action.

**Don't** use the "text" usage as a high-priority action.

### Sizing

**Do** use the default size for all use cases where possible of the Button.

**Don't** use the "m" size _unless_ it is a **last resort**.

## Accessbility

The Pando Icon Button, and Icon APIs combined with semantic HTML allow the Icon Button to be fully accessible and screen-readable.
