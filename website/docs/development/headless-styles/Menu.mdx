### Submenus

When you need a submenu, set `kind` to `submenu` and add your `label` options.
When a submenu is displayed, the `aria-expanded` on the triggering element should be set to `"true"`.

## Accessibility

The `label` option provides a non-visual label for your submenu that describes its contents for assistive technologies.

### Hover vs Click

Opening and closing a menu is an intentional action and should therefore trigger on click or with keyboard navigation.
Hover should be avoided since it does not convey intent and is difficult for some people and devices (touch screens).

### Keyboard navigation

Menus must provide [keyboard support](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Roles/menu_role#keyboard_interactions) to navigate all items in the menu.
Keyboard support for a menu typically include the arrow keys to navigate between the menu items and open/close submenus.

The preferable implementation is to use a
[roving tabindex](https://developer.mozilla.org/en-US/docs/Web/Accessibility/Keyboard-navigable_JavaScript_widgets#technique_1_roving_tabindex)
for managing focus and tab order.

:::tip

We provide hooks for [menu](../react-utils/use-menu-interaction.mdx) and [submenu](../react-utils/use-submenu-interaction.mdx) interactions, as well as [roving tabindex](../react-utils/use-roving-tabindex.mdx) focus management, as part of our react-utils package.

:::

## API

| Name             | Argument Type                            | Default                                                              | Description                         |
| ---------------- | ---------------------------------------- | -------------------------------------------------------------------- | ----------------------------------- |
| `getMenuProps`   | **options**: [MenuOptions](#menuoptions) | **isExpanded**: false <br />**kind**: 'menu' <br/> **label**: 'menu' | Get menu styles and a11y properties |
| `getJSMenuProps` | **options**: [MenuOptions](#menuoptions) | same as CSS API                                                      | CSS-in-JS API                       |

## Options

### MenuOptions

This interface is available as an `import` for use.

```typescript
export interface MenuOptions {
  isExpanded?: boolean
  kind?: 'menu' | 'submenu'
  label?: string
  tech?: Tech
}
```

```typescript title="Import example"
import type { MenuOptions } from '@pluralsight/headless-styles/types'
```

## Props

### MenuProps

```typescript
interface MenuProps {
  wrapper: {
    className: string
  }
  trigger: {
    'aria-expanded': boolean
    'aria-haspopup': boolean
  }
  menu: {
    'aria-label': string
    className: string
    'data-expanded': boolean
    role: 'menu'
  }
  menuListItem: {
    className: string
    role: 'presentation'
  }
  menuItem: {
    'aria-expanded'?: boolean
    'aria-haspopup'?: boolean
    className: string
    role: 'menuitem'
    tabIndex: -1
  }
  iconOptions?: {
    ariaHidden: true
    className: string
    size: 'l'
    tech: 'svelte' | ''
  }
}
```
