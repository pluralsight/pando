## Import

```javascript
import {
  getButtonProps,
  getDangerButtonProps,
} from '@pluralsight/headless-styles'
```

- **getButtonProps**: The styles for any button outside of status related
- **getDangerButtonProps**: The styles for danger status buttons

## Usage

`getButtonProps` comes with five kinds to change the visual style of a button. You can use `text` (default), `textWeak`, `weak`, `medium`, and `strong`.

### Button styled links

If you want to use button styles for an `a` element, just omit the `type` property from the return Object sent.

```jsx
const { className } = getButtonProps()
const psButtonLinkProps = { className: psButtonProps.className }

function BackLink() {
  return (
    <a href="#" {...psbButtonLinkProps}>
      Back
    </a>
  )
}
```

:::tip

With ES\* you could destructure the className out of the object (i.e. `{ className, ...rest }`). However, this approach forces you to declare an unused variable which will negatively impact performance at some point, make a negative impact on code readability, and fail sonarlint tests.

:::

## CSS API

### getButtonProps

```typescript
getButtonProps(options: ButtonOptions): ButtonProps
```

The button prop-getter takes in [ButtonOptions](#buttonoptions) returns [ButtonProps](#buttonprops) that contains all the a11y attributes needed along with all the styles for you to use in any fashion you like - or easily extend/overwrite when needed.

### getDangerButtonProps

```typescript
getDangerButtonProps(options: DangerButtonOptions): ButtonProps
```

The danger button prop-getter takes in [DangerButtonOptions](#dangerbuttonoptions) returns [ButtonProps](#buttonprops) that match the `getButtonProps` API.

## CSS in JS API

### getJSButtonProps

```typescript
getJSButtonProps(options: ButtonOptions): JSButtonProps
```

The CSS-in-JS prop-getter takes in the same [ButtonOptions](#buttonoptions) as the CSS API and returns [JSButtonProps](#jsbuttonprops) which caters to CSS Templates, StyleObject, and button specific attributes.

### getJSDangerButtonProps

```typescript
getJSDangerButtonProps(options: DangerButtonOptions): JSButtonProps
```

The CSS-in-JS danger button prop-getter takes in [DangerButtonOptions](#dangerbuttonoptions) and returns [JSButtonProps](#jsbuttonprops).

## Extending

There are times you may need to make a slight adjustment to the button styles which is easy to do since we are giving you an Object.

Here is an example of using `styled-components` to extend a button text color.

```jsx title=page/Login/components/CustomButton.jsx
import styled from 'styled-components'
import { getJSButtonProps } from '@pluralsight/headless-styles'

const CustomButton = styled.button`
  ${getJSButtonProps({ kind: 'medium' }).cssProps}
  color: '#bfbfbf'
`

export default CustomButton
```

:::tip

When combining `headless-styles` with other libraries, please follow the guides in the example codesandboxes for the best experience. **Some libraries require special support**.

:::

## API

| Name                     | Argument Type                                            | Default                               | Description               |
| ------------------------ | -------------------------------------------------------- | ------------------------------------- | ------------------------- |
| `getButtonProps`         | **options**: [ButtonOptions](#buttonoptions)             | **kind**: 'text', <br />**size**: 'm' | Get main button styles.   |
| `getDangerButtonProps`   | **options**: [DangerButtonOptions](#dangerbuttonoptions) | **kind**: 'text', <br />**size**: 'm' | Get danger button styles. |
| `getJSButtonProps`       | **options**: [ButtonOptions](#buttonoptions)             | **kind**: 'text', <br />**size**: 'm' | Get main button styles.   |
| `getJSDangerButtonProps` | **options**: [DangerButtonOptions](#dangerbuttonoptions) | **kind**: 'text', <br />**size**: 'm' | Get danger button styles. |

## ButtonOptions

```typescript
interface ButtonOptions
  extends DetailedHTMLProps<
    ButtonHTMLAttributes<HTMLButtonElement>,
    HTMLButtonElement
  > {
  kind?: 'text' | 'text-weak' | 'weak' | 'medium' | 'strong'
  size?: 'xs' | 's' | 'm' | 'l'
  tech?: 'svelte'
}
```

## DangerButtonOptions

```typescript
interface DangerButtonOptions extends ButtonOptions {
  kind?: 'text' | 'medium' | 'strong'
  size?: 'xs' | 's' | 'm' | 'l'
  tech?: 'svelte'
}
```

interface ButtonProps {

## JSButtonProps

```typescript
interface JSButtonProps {
  cssProps: TemplateLiteralStringArray | string
  styles: Record<CSSProps, string>
  type: ButtonType
}
```

```js title="Return value example"
{
  cssProps: `
    color: #fff;
    background-color: #000;
    ...
  `,
  styles: {
    color: '#fff',
    backgroundColor: '#000'
    '&:hover': {
      ...
    }
  },
  type: 'button',
}
```

## Button size guide

Button sizes are relative to the `padding` and `font-size` styles. Here is an interface that gives the sizing guide. Variables are referenced from the [`tokens`](../tokens/intro.md) package.

```typescript title="Example interface - does not exist in types"
interface ButtonSizeGuide {
  xs: {
    fontSize: '0.75rem'
    paddingInlineStart: '0.5rem'
    paddingInlineEnd: '0.5rem'
  }
  s: {
    fontSize: '0.875rem'
    paddingInlineStart: '0.75rem'
    paddingInlineEnd: '0.75rem'
  }
  m: {
    fontSize: '1rem'
    paddingInlineStart: '1rem'
    paddingInlineEnd: '1rem'
  }
  l: {
    fontSize: '1rem'
    paddingInlineStart: '1.5rem'
    paddingInlineEnd: '1.5rem'
  }
}
```

## Button color guide

Button colors are relative to the `background` style and have a direct correlation from the `kind` to the [`design-tokens`](../tokens/intro.md) package. Here is an interface that gives the color guide.

```typescript title="Example interface - does not exist in types"
interface ButtonColorGuide {
  text: 'transparent'
  'text-weak': 'transparent'
  weak: 'var(--ps-neutral-background)'
  medium: 'var(--ps-background)'
  strong: 'var(--ps-background-weak)'
}

interface DangerButtonColorGuide {
  text: 'transparent'
  medium: 'var(--ps-danger-background)'
  strong: 'var(--ps-danger-background-weak)'
}
```
