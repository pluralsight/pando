## API

| Name              | Argument Type                              | Default                                                            | Description       |
| ----------------- | ------------------------------------------ | ------------------------------------------------------------------ | ----------------- |
| `getBadgeProps`   | **options**: [BadgeOptions](#badgeoptions) | **sentiment**: 'default'<br />**usage:** filled<br />**size:** 's' | Get badge styles. |
| `getJSBadgeProps` | **options**: [BadgeOptions](#badgeoptions) | same as CSS API                                                    | CSS-in-JS API.    |

## Options

### BadgeOptions

This interface is available as an `import` for use.

```typescript
interface BadgeOptions {
  sentiment?: 'default' | 'action'
  usage?: 'filled' | 'outline'
  size?: 'xs' | 's'
}
```

```typescript title="Import example"
import type { BadgeOptions } from '@pluralsight/headless-styles/types'
```

## Props

### BadgeProps

```typescript
interface BadgeProps {
  iconOptions: {
    ariaHidden: boolean
    ariaLabel: string
    size: string
  }
  badge: {
    className: string
  }
  iconWrapper: {
    className: string
  }
}
```

### JSBadgeProps

```typescript
interface JSBadgeProps {
  iconOptions: {
    ariaHidden: boolean
    ariaLabel: string
    size: string
  }
  badge: {
    cssProps: string
    styles: Record<CSSProps, string>
  }
  iconWrapper: {
    cssProps: string
    styles: Record<CSSProps, string>
  }
}
```

## Guides

### Badge color guide

Badge colors are relative to the `background` style and have a direct correlation from the `sentiment` to the [`design-tokens`](../tokens/intro.md) package. Here is an interface that gives the color guide.

```typescript title="Example interface - does not exist in types"
interface BadgeColorGuide {
  default: 'var(--ps-surface-medium)'
  action: 'var(--ps-action-background)'
  outline: 'transparent'
}
```
