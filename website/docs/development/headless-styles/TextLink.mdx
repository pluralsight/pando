## Accessible Link Text

When using assistive technology, some users will jump directly to links.
Therefore, it is important the link text clearly communicates its purpose without context (i.e. avoid phrases like “Click Here,” “More", “Read More”, etc.).

## API

| Name                 | Argument Type                                    | Default | Description           |
| -------------------- | ------------------------------------------------ | ------- | --------------------- |
| `getTextLinkProps`   | **options**: [TextLinkOptions](#textlinkoptions) |         | Get main link styles. |
| `getJSTextLinkProps` | **options**: [TextLinkOptions](#textlinkoptions) |         | CSS-in-JS API         |

## Options

### TextLinkOptions

This interface is available as an `import` for use.

```typescript
interface TextLinkOptions {
  href: string
  tech?: 'svelte'
}
```

```typescript title="Import example"
import type { TextLinkOptions } from '@pluralsight/headless-styles/types'
```

## Props

### TextLinkProps

```typescript
interface TextLinkProps {
  link: {
    href: string
    className: string
    rel?: 'noopener external'
    target?: '_blank'
  }
  iconOptions: {
    ariaHidden: false
    ariaLabel: '(opens in a new window)'
    customSize: '1em'
    tech: '' | 'svelte'
  }
}
```

### JSTextLinkProps

```typescript
interface JSTextLinkProps {
  link: {
    href: string
    rel?: 'noopener external'
    target?: '_blank'
  }
  iconOptions: {
    ariaHidden: false
    ariaLabel: '(opens in a new window)'
    customSize: '1em'
  }
  cssProps: TemplateLiteralStringArray | string
  styles: Record<CSSProps, string>
}
```
