## Import

```javascript
import { getIconProps } from '@pluralsight/headless-styles'
```

- **getIconProps**: The styles and a11y attributes for any SVG icon

## Usage

### Icon sizes

Use the `size` options to change the size of an icon. The options are: `s`, `m` (default), `l`.

## Accessibility

An icon alone does not convey any useful information to non-visual users.
Depending on the usage, you will need to provide additional information.

### Decorative icons

The icon is purely cosmetic and conveys no real semantic meaning.
In this case, the icon should be hidden from assistive technology.

```javascript
const briefcaseIconProps = getIconProps({ ariaHidden: true })
```

### Interactive icons

SVG elements are not semantically interactive elements;
when the user would interact with an icon, a more appropriate element should "own" the interaction:

- The icon element should be a child of a `<button>` or `<a>` element.
- The `<button>` or `<a>` should have a meaningful label provided either through direct text content, `aria-label`, or `aria-labelledby`.

See the [Button wth icon](./Button.mdx#button-with-icon) and [Button icon (no label)](./Button.mdx#button-icon-no-label) examples.

### Indicator icons

When an icon visually communicates some information (such as a status), that information must also be made available to screen readers.
Use the `label` option to add a non-visible, accessible label to the icon.

```javascript
const volumeOffIconProps = getIconProps({ label: 'Volume is off' })
```

## CSS API

### getIconProps

```typescript
getIconProps(options: IconOptions): IconProps
```

The icon prop-getter takes in [IconOptions](#iconoptions) returns [IconProps](#iconprops) that contains all the a11y attributes needed along with all the styles for you to use in any fashion you like - or easily extend/overwrite when needed.

## JS API

### getJSIconProps

```typescript
getJSIconProps(options: IconOptions): JSIconProps
```

the CSS-in-JS prop-getter takes in the same [IconOptions](#iconoptions) as the CSS API and returns [JSIconProps](#jsiconprops) which caters to CSS Templates, StyleObject, and a11yProps.

## API

| Name           | Argument Type                            | Default       | Description           |
| -------------- | ---------------------------------------- | ------------- | --------------------- |
| `getIconProps` | **options**: [IconOptions](#iconoptions) | **size**: 'm' | Get main icon styles. |

## IconOptions

```typescript
export interface IconOptions extends IconA11yOptions {
  size?: 's' | 'm' | 'l'
  tech?: 'svelte'
}
```

## IconA11yOptions

```typescript
export interface IconA11yOptions {
  label?: string
  ariaHidden?: boolean
}
```

## IconA11yProps

```typescript
interface IconA11yProps {
  'aria-label': string
  'aria-hidden': boolean
  role: 'img'
}
```

## IconProps

```typescript
interface IconProps {
  className: string
  ...IconA11yProps
}
```

## JSIconProps

```typescript
interface IconProps {
  a11yProps: IconA11yProps
  cssProps: TemplateLiteralArray
  styles: Record<string, unknown>
}
```
