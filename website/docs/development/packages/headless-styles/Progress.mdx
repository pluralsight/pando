## Import

```javascript
import { getProgressProps } from '@pluralsight/headless-styles'
```

## Usage

### Basic Progress

`getProgressProps` comes with two kinds to change the visual style of the progress. You can use `linear` (default), and `inset`.

## Accessibility

`getProgressProps` returns an [a11yProps](#a11yprops) Object to spread over the "bar" component.

- It sets the `role` to `progressbar` to denote that it is a progress.
- It uses `aria-valuenow` set to the `now` value passed into the [ProgressOptions](#progressoptions) to ensure the progress percent is visible to screen readers.

## CSS API

### getProgressProps

```typescript
getProgressProps(options: ProgressOptions): ProgressProps
```

The progress prop-getter takes in [ProgressOptions](#progressoptions) returns [ProgressProps](#progressprops) that contains the a11y attributes needed along with all the styles for you to use in any fashion you like - or easily extend/overwrite when needed.

## JS API

### getJSProgressProps

```typescript
getJSProgressProps(options: ProgressOptions): JSProgressProps
```

The CSS-in-JS prop-getter takes in the same [ProgressOptions](#progressoptions) as the CSS API and returns [JSProgressProps](#jsprogressprops) which caters to CSS Templates, StyleObject, and button specific attributes.

:::note

The custom bar `width` percentage is added into the `cssProps` and `styles` for the JS API.

:::

## API

| Name                 | Argument Type                                    | Default                                 | Description               |
| -------------------- | ------------------------------------------------ | --------------------------------------- | ------------------------- |
| `getProgressProps`   | **options**: [ProgressOptions](#progressoptions) | **kind**: 'linear', <br />**size**: 's' | Get main progress styles. |
| `getJSProgressProps` | **options**: [ProgressOptions](#progressoptions) | **kind**: 'linear', <br />**size**: 's' | Get main progress styles. |

## ProgressOptions

```typescript
interface ProgressOptions extends ProgressA11yOptions {
  kind?: 'linear' | 'inset'
  size?: 'xs' | 's'
  tech?: 'svelte'
}
```

## ProgressA11yOptions

```typescript
interface ProgressA11yOptions {
  max?: number
  min?: number
  now?: number
}
```

## A11yProps

```typescript
interface A11yProps {
  'aria-valuemax': number
  'aria-valuemin': number
  'aria-valuenow': number
  role: 'progressbar'
}
```

## ProgressProps

```typescript
interface ProgressProps {
  a11yProps: A11yProps
  style: {
    width: number
  }
  classes: {
    bar: { className: string }
    wrapper: { className: string }
  }
}
```

## JSProgressProps

```typescript
interface JSProgressProps {
  a11yProps: A11yProps
  bar: {
    cssProps: TemplateLiteralArray
    styles: Record<string, unknown>
  }
  wrapper: {
    cssProps: TemplateLiteralArray
    styles: Record<string, unknown>
  }
}
```
