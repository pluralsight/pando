## Anatomy

2. External Icon (optional)

## Usage

### Basic Links

```jsx live
function BasicTextLinks() {
  const gridItem = getGridItemProps({ colSpan: 2 })
  const externalLink = getTextLinkProps({
    href: 'https://github.com/pluralsight/pando',
  })

  return (
    <div
      data-site-override="alignGridCenter"
      {...getGridProps({ cols: 12, gap: 16 })}
    >
      <div {...gridItem}>
        <a {...getTextLinkProps({ href: '#' }).link}>Local Text Link</a>
      </div>

      <div {...gridItem}>
        <a {...externalLink.link}>
          External Link
          <ExternalLinkIcon {...getIconProps(externalLink.iconOptions)} />
        </a>
      </div>
    </div>
  )
}
```

## Reference

```typescript
getTextLinkProps(options?: TextLinkOptions)
```

### Parameters

- `options`: A String value for the property href.

### Returns

An Object matching the [Anatomy](#anatomy) containing `className` and a11y properties relevant to each element/section. Additionally, it returns an `iconOptions` Object to use as the parameters for the Icon API.

## Behavior

### Usage

**Do** use the Text Link when you need to provide a way to navigate to a different page.

**Don't** use a Button to navigate the user to a different page.

**Do** use the ExternalLink Icon when you have a link that will navigate the user outside of your application/site.

**Don't** eliminate the ExternalLink Icon _unless_ the link uses a local path.

## Accessbility

The Pando Text Link, and Icon APIs combined with semantic HTML allow the Text Link to be fully accessible and screen-readable.
